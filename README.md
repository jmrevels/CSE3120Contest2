# CSE3120 Contest 2
## Author: Joshua Revels


# Project Description
For my second project, I created an interpretation of Whack-a-Mole. This program uses popup windows to represent the "mole," and gives a score based on how quickly the mole was "whacked" (clicked). 
Once a mole is whacked, the window will close and a new one will appear in its place.
The number of moles is limited to 10 in order to keep the score consistent across runs.


# Compilation Instructions
The program should compile either by using the VS Code debug environment or by using the asm_CSE3120.bat created at the start of the semester. Irvine32.inc and GraphWin.inc are required (should both be included in the Irvine folder).

Note: It is suggested (but not required) to close out of other windows before running this program, as it involves creating other windows.
